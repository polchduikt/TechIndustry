# Фінальний проєкт: Security Hardening

---

## 🎯 Мета уроку
- зрозуміти фундамент теми та її роль у системі захисту
- навчитися приймати рішення на основі ризику
- підготувати практичний baseline для власного проєкту

---

## 🧠 Теорія
Фінальний модуль обєднує попередні практики в єдиний baseline для сервісу. Мета не в ідеальній теорії, а в практичному наборі контролів, які реально працюють у вашому процесі доставки. Результат має бути перевірюваним через метрики та регулярні ревю.

### Поглиблено
Безпека ефективна тоді, коли процеси повторювані: є політика, автоматизація перевірок, власники контролів і регулярний перегляд ризиків. Одноразові дії без системного супроводу швидко втрачають ефект.

### Приклад 1
Deliverables: threat model, security checklist, CI security stage, incident runbook.

### Приклад 2
Критерії готовності: critical findings нуль, MFA ввімкнено для адміністраторів, audit logs централізовані.

---

## 💻 Практичний приклад
Вибери один сервіс вашого проєкту і підготуй короткий план посилення безпеки за темою уроку: що змінити в найближчий спринт, які ризики це зменшить, як перевірити результат.

### Міні завдання
- опиши 3 конкретні ризики
- запропонуй 3 контролі
- вкажи метрику перевірки для кожного контролю

---

## 📝 Підсумки
- кібербезпека будується шарами і процесами
- контроль має бути вимірюваним та перевірюваним
- стабільний результат дає регулярність, а не разова активність

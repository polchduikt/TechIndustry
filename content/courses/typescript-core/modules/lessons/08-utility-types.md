# Utility типи

Utility типи в TypeScript дозволяють швидко створювати похідні моделі на основі базових інтерфейсів.

---

## 🎯 Мета уроку
Після цього уроку ти зможеш:
- використовувати `Partial`, `Required`, `Pick`, `Omit`
- формувати DTO без дублювання
- готувати типи для форм і API
- оптимізувати підтримку моделей

---

## 🧠 Теорія

### Найчастіші utility типи
```ts
interface User {
  id: number;
  name: string;
  email: string;
}

type UserUpdate = Partial<User>;
type UserStrict = Required<User>;
```

---

### Вибір і виключення полів
```ts
type UserPublic = Omit<User, "email">;
type UserName = Pick<User, "name">;
```

---

### Навіщо це в реальному коді
Один базовий тип -> багато спеціалізованих представлень без повторного опису структури.

---
### Глибше про керування моделями
Utility-типи корисні там, де одна сутність має кілька представлень: створення, оновлення, публічна відповідь, внутрішня модель. Вони дозволяють синхронізувати ці представлення з єдиним джерелом правди.

Це зменшує ризик ситуацій, коли поля роз'їжджаються між DTO і доменною моделлю після чергового рефакторингу.

## 💻 Практичний приклад

```ts
interface Course {
  id: number;
  title: string;
  description: string;
  isPublished: boolean;
}

type CourseCreateDto = Pick<Course, "title" | "description">;
type CoursePatchDto = Partial<Omit<Course, "id">>;
```

---

## 📝 Підсумки
- utility типи підвищують швидкість розробки
- менше ручного дублювання = менше помилок
- моделі стають більш послідовними між модулями

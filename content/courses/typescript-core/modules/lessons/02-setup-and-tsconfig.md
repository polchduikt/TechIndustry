# Налаштування TypeScript і tsconfig

Початок роботи з TypeScript складається з двох кроків: встановлення компілятора і правильного налаштування `tsconfig.json`.

---

## 🎯 Мета уроку
Після цього уроку ти зможеш:
- встановити TypeScript у проєкт
- згенерувати і налаштувати `tsconfig`
- розуміти ключові compiler options
- запускати компіляцію і перевірку типів

---

## 🧠 Теорія

### Встановлення
```bash
npm i -D typescript
npx tsc --init
```

---

### Важливі параметри
- `target`
- `module`
- `strict`
- `outDir`
- `rootDir`

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "CommonJS",
    "strict": true,
    "rootDir": "src",
    "outDir": "dist"
  }
}
```

---

### Режими роботи
- одноразова перевірка: `npx tsc`
- watch-режим: `npx tsc -w`

---
### Глибше про конфігурацію проєкту
`tsconfig` визначає не лише технічні параметри компіляції, а й стандарти коду команди. Непослідовні налаштування між пакетами або сервісами часто створюють складні для діагностики помилки на інтеграції.

Тому варто мати базовий загальний конфіг і наслідувати його в підпроєктах, щоб зберігати єдині правила перевірки типів.

## 💻 Практичний приклад

```ts
const env: "dev" | "prod" = "dev";
console.log(`Mode: ${env}`);
```

---

## 📝 Підсумки
- коректний `tsconfig` формує якість проєкту
- `strict` бажано вмикати з першого дня
- автоматична перевірка типів прискорює розробку

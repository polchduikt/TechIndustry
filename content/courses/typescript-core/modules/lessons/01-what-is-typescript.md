# Що таке TypeScript

TypeScript — це надмножина JavaScript, яка додає типізацію, контракти і кращий контроль якості коду ще до запуску застосунку.

---

## 🎯 Мета уроку
Після цього уроку ти зможеш:
- пояснити, чому TypeScript потрібен у великих проєктах
- назвати ключові переваги над чистим JavaScript
- розуміти, як відбувається компіляція `.ts` у `.js`
- описати реальні сценарії застосування TS

---

## 🧠 Теорія

### Переваги TypeScript
- раннє виявлення помилок
- безпечніший рефакторинг
- краща підтримка IDE

```ts
const title: string = "TypeScript Core";
```

---

### Компіляція
TypeScript-компілятор перевіряє типи й перетворює код у JavaScript, який виконується в runtime.

```ts
function sum(a: number, b: number): number {
  return a + b;
}
```

---

### Де TS дає максимум користі
- enterprise frontend
- backend-сервіси
- SDK та бібліотеки

---
### Глибше про роль типів у команді
У командній розробці типи — це форма документації, яка перевіряється автоматично. Вони зменшують кількість непорозумінь між frontend, backend і QA, бо очікування щодо даних закріплені в коді.

Типізація також пришвидшує рев'ю: замість здогадок про структуру даних рев'юер одразу бачить контракт і може зосередитись на бізнес-логіці.

## 💻 Практичний приклад

```ts
interface Course {
  id: number;
  name: string;
}

const course: Course = { id: 1, name: "TS Basics" };
console.log(course);
```

---

## 📝 Підсумки
- TypeScript зменшує кількість runtime-помилок
- типізація покращує якість командної розробки
- TS особливо корисний для масштабованих проєктів

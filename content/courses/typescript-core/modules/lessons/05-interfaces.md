# Інтерфейси

Інтерфейси в TypeScript визначають форму об'єктів і допомагають тримати архітектуру послідовною.

---

## 🎯 Мета уроку
Після цього уроку ти зможеш:
- створювати інтерфейси для моделей даних
- використовувати optional поля
- розширювати інтерфейси через `extends`
- застосовувати інтерфейси в сервісах і функціях

---

## 🧠 Теорія

### Базовий інтерфейс
```ts
interface User {
  id: number;
  name: string;
  email?: string;
}
```

---

### Наслідування інтерфейсів
```ts
interface Admin extends User {
  permissions: string[];
}
```

---

### Інтерфейси для функцій
```ts
interface Formatter {
  (value: string): string;
}

const upper: Formatter = (value) => value.toUpperCase();
```

---
### Глибше про інтерфейси в архітектурі
Інтерфейси добре працюють як межа між шарами: контролер знає лише контракт сервісу, а не конкретну реалізацію. Це дозволяє легко підміняти реалізації для тестів, staging-середовища або міграції інфраструктури.

Коли контракти стабільні, внутрішня реалізація може змінюватися значно швидше без ризику зламати зовнішніх споживачів.

## 💻 Практичний приклад

```ts
interface Course {
  id: number;
  title: string;
  lessons: number;
}

function printCourse(course: Course): void {
  console.log(`${course.title} (${course.lessons} уроків)`);
}

printCourse({ id: 1, title: "TypeScript Core", lessons: 10 });
```

---

## 📝 Підсумки
- інтерфейси фіксують контракт між частинами коду
- `extends` допомагає масштабувати моделі
- типізовані контракти знижують ризик інтеграційних помилок

# Встановлення та інструменти Kotlin

Щоб працювати з Kotlin комфортно, важливо правильно зібрати середовище: JDK, IDE, система збірки та базові команди запуску.

---

## 🎯 Мета уроку
Після цього уроку ти зможеш:
- встановити мінімально необхідні інструменти
- створити Kotlin-проєкт у IntelliJ IDEA
- орієнтуватись у `build.gradle.kts`
- запускати і збирати проєкт через Gradle

---

## 🧠 Теорія

### Мінімальний набір
1. JDK 17+
2. IntelliJ IDEA
3. Kotlin plugin (зазвичай вбудований)
4. Gradle

```kotlin
fun main() {
    println("Environment ready")
}
```

---

### Роль Gradle
Gradle керує залежностями, збіркою, тестами та запуском. Для Kotlin часто використовують Kotlin DSL (`build.gradle.kts`).

```kotlin
plugins {
    kotlin("jvm") version "2.0.0"
    application
}
```

---

### Структура проєкту
- `src/main/kotlin` — основний код
- `src/test/kotlin` — тести
- `build.gradle.kts` — конфігурація

```kotlin
application {
    mainClass.set("MainKt")
}
```

---
### Глибше про стабільне середовище
Проблеми з інструментами часто маскуються під проблеми коду: різні версії JDK, плагінів або Gradle можуть давати різний результат збірки. Тому важливо фіксувати версії, використовувати один стандартний конфіг для всієї команди та перевіряти збірку в CI.

Якщо структура проєкту та збірка налаштовані правильно з початку, додавання модулів, тестів і нових залежностей стає передбачуваним процесом.

## 💻 Практичний приклад

```kotlin
fun sum(a: Int, b: Int): Int = a + b

fun main() {
    val result = sum(4, 7)
    println("Результат: $result")
}
```

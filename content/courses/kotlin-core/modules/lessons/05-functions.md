# Функції в Kotlin

Функції в Kotlin дозволяють описувати логіку компактно і без дублювання, особливо завдяки параметрам за замовчуванням та іменованим аргументам.

---

## 🎯 Мета уроку
Після цього уроку ти зможеш:
- оголошувати функції з параметрами і типом повернення
- використовувати default-параметри
- застосовувати named arguments
- створювати однорядкові функції

---

## 🧠 Теорія

### Базовий синтаксис
```kotlin
fun add(a: Int, b: Int): Int {
    return a + b
}
```

---

### Параметри за замовчуванням
```kotlin
fun greet(name: String, prefix: String = "Привіт"): String {
    return "$prefix, $name"
}
```

---

### Іменовані аргументи
```kotlin
val message = greet(name = "Ірина", prefix = "Вітаю")
println(message)
```

---
### Глибше про дизайн функцій
Якісна функція має чітку відповідальність: одна функція — одна логічна дія. Якщо функція одночасно валідує, перетворює і зберігає дані, її важко підтримувати й тестувати.

Краще розділяти кроки на менші функції, а потім комбінувати їх у вищому рівні сервісу. Це підвищує перевикористання та зменшує кількість регресій.

## 💻 Практичний приклад

```kotlin
fun createOrder(product: String, quantity: Int = 1, urgent: Boolean = false): String {
    val type = if (urgent) "термінове" else "звичайне"
    return "Замовлення: $product x$quantity ($type)"
}

fun main() {
    println(createOrder("Книга Kotlin"))
    println(createOrder(product = "Курс", quantity = 2, urgent = true))
}
```

---

## 📝 Підсумки
- функції з default-параметрами зменшують шаблонність
- named arguments покращують читабельність викликів
- короткі функції легше тестувати і перевикористовувати

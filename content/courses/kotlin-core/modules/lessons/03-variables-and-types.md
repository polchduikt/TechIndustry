# Змінні та типи даних у Kotlin

Типізація у Kotlin строга, але гнучка: багато типів виводяться автоматично, а код залишається безпечним.

---

## 🎯 Мета уроку
Після цього уроку ти зможеш:
- впевнено використовувати `val` та `var`
- працювати з базовими типами Kotlin
- застосовувати інтерполяцію рядків
- виконувати конвертацію значень між типами

---

## 🧠 Теорія

### `val` і `var`
- `val` — незмінне посилання
- `var` — змінне посилання

```kotlin
val name = "TechIndustry"
var users = 10
users += 5
```

---

### Базові типи
- `Int`, `Long`, `Double`, `Float`
- `Boolean`, `Char`, `String`

```kotlin
val price: Double = 199.99
val active: Boolean = true
```

---

### Рядки та конвертація
```kotlin
val amountText = "42"
val amount = amountText.toInt()
println("Кількість: $amount")
```

---
### Глибше про типобезпеку
Типи в Kotlin виконують роль контракту: вони не лише описують дані, а й підказують, що з цими даними можна робити безпечно. Чим точніше обрано тип, тим менше прихованих помилок під час рефакторингу.

У великих модулях корисно явно вказувати тип там, де бізнес-сенс неочевидний. Це покращує читабельність і пришвидшує онбординг нових розробників.

## 💻 Практичний приклад

```kotlin
fun main() {
    val product = "Kotlin Course"
    val unitPrice = 120.0
    val quantity = 3
    val total = unitPrice * quantity

    println("Товар: $product")
    println("Сума: $total")
}
```

---

## 📝 Підсумки
- правильний вибір типів зменшує кількість помилок
- `val` варто використовувати за замовчуванням
- явна типізація корисна в критичних ділянках коду

# Inheritance and Polymorphism

Програмісти дуже ліниві (у хорошому сенсі цього слова). Вони ненавидять писати один і той самий код двічі. ООП дає нам суперсилу, яка дозволяє класам "успадковувати" властивості інших класів, як діти успадковують колір очей від батьків.

---

## Мета уроку

Після цього уроку ти зможеш:

- розуміти, що таке Спадкування та як воно економить час
- використовувати ключове слово `extends` для створення підкласів
- розуміти, що таке Поліморфізм (багатоликість)
- перевизначати методи батьківського класу за допомогою `@Override`

---

## Теорія

### Що таке Спадкування (Inheritance)?

Уяви, що ти розробляєш гру. У тебе є Собака, Кіт і Вовк. Всі вони вміють дихати, ходити і їсти. Замість того, щоб писати методи `eat()` і `walk()` три рази для кожної тварини окремо, ми створюємо один загальний клас **`Animal`** (Тварина).

Потім ми кажемо, що `Dog` (Собака) **розширює** (успадковує) клас `Animal`. Клас `Animal` стає "батьком" (Superclass), а `Dog` — "дитиною" (Subclass). Дитина автоматично отримує всі здібності батька без написання зайвого коду!

Для цього в Java використовується слово **`extends`**.

### Що таке Поліморфізм?

"Полі" означає "багато", а "морф" — "форм". Поліморфізм — це здатність різних об'єктів реагувати на одну й ту саму команду по-різному.

Наприклад, у нас є команда `makeSound()` (видати звук).
Якщо ми викличемо її для Собаки — вона скаже "Гав". Якщо для Кота — він скаже "Мяу". Команда однакова, але поведінка різна!
Щоб змінити поведінку батьківського методу, дитина використовує "перевизначення" — пише над своїм методом спеціальну мітку **`@Override`**.

## Простий приклад

```java
// БАТЬКІВСЬКИЙ КЛАС
class Enemy {
    String name;

    public void attack() {
        System.out.println(name + " атакує базовим ударом!");
    }
}

// КЛАС-ДИТИНА (Зомбі успадковує все від Ворога)
class Zombie extends Enemy {
    
    // Зомбі має свою унікальну здатність, якої немає у звичайних ворогів
    public void bite() {
        System.out.println(name + " намагається вкусити!");
    }

    // ПОЛІМОРФІЗМ: Зомбі атакує не так, як звичайний ворог.
    // Ми "перевизначаємо" батьківський метод
    @Override
    public void attack() {
        System.out.println(name + " повільно махає руками!");
    }
}

// КЛАС-ДИТИНА (Дракон теж успадковує Ворога)
class Dragon extends Enemy {
    
    @Override
    public void attack() {
        System.out.println(name + " дихає вогнем на весь екран!!!");
    }
}

public class GameDemo {
    public static void main(String[] args) {
        Zombie myZombie = new Zombie();
        myZombie.name = "Повільний Зомбі"; // Поле name успадковано від Enemy!
        
        Dragon myDragon = new Dragon();
        myDragon.name = "Чорний Дракон";
        
        // Викликаємо однакову команду, але отримуємо різний результат (Поліморфізм)
        myZombie.attack();
        myDragon.attack();
        
        // Викликаємо унікальну здатність зомбі
        myZombie.bite();
    }
}

# Methods and Scope

Уяви, що ти граєш у гру, і твоєму персонажу потрібно зібрати 10 монет. Замість того, щоб 10 разів писати команду "зроби крок, нахилися, візьми монету, поклади в інвентар", набагато простіше створити одну кнопку "Зібрати монету" і просто натискати її. У програмуванні такі "кнопки" називаються методами!

---

## Мета уроку

Після цього уроку ти зможеш:

- розуміти, що таке метод і для чого він потрібен
- створювати власні методи та викликати їх
- передавати в методи дані (параметри) та отримувати результат
- розуміти, що таке "область видимості" змінних (Scope)

---

## Теорія

### Що таке метод?

**Метод** — це іменований блок коду, який виконує певну дію. Це ніби міні-програма всередині твоєї великої програми. Ми використовуємо методи, щоб не писати один і той самий код багато разів (це правило називається DRY — Don't Repeat Yourself, або "Не повторюйся").

### Як створити метод?

Кожен метод має кілька важливих частин:
1. **Тип повернення:** Що метод нам віддасть після своєї роботи? Якщо це число, ми пишемо `int`. Якщо текст — `String`. Якщо метод нічого не повертає, а просто виконує дію (наприклад, виводить текст на екран), ми пишемо слово `void` (порожнеча).
2. **Ім'я методу:** Називай методи так, щоб було зрозуміло, що вони роблять. Зазвичай це дієслово: `calculateDamage()`, `printName()`, `jump()`.
3. **Параметри (аргументи):** Це дані, які ми передаємо всередину методу для роботи (в дужках). Наприклад, щоб приготувати чай, методу `makeTea()` потрібні параметри: кількість цукру та тип заварки.

### Що таке Область видимості (Scope)?

**Scope** — це правило, яке визначає, де твоя змінна "живе" і де її можна використовувати.
Уяви, що фігурні дужки `{ }` — це стіни кімнати. Якщо ти створюєш змінну всередині методу або всередині циклу (в цій "кімнаті"), то зовні про неї ніхто не знатиме! Коли метод закінчує свою роботу, всі змінні, створені всередині нього, просто зникають.

## Простий приклад

Давай створимо програму для підрахунку шкоди (damage) в грі за допомогою методів:

```java
public class MethodsDemo {

    // Головний метод, з якого починається гра
    public static void main(String[] args) {
        System.out.println("Гра почалася!");
        
        // Викликаємо наш метод і передаємо йому числа
        int damage1 = calculateDamage(50, 2);
        System.out.println("Герой наніс " + damage1 + " одиниць шкоди!");
        
        int damage2 = calculateDamage(30, 1); // Використовуємо метод знову!
        System.out.println("Герой наніс " + damage2 + " одиниць шкоди!");
        
        // Викликаємо метод, який нічого не повертає (void)
        showGameOverScreen();
    }

    // Наш власний метод, який приймає базову атаку і множник критичного удару
    // Він повертає ціле число (int)
    public static int calculateDamage(int baseAttack, int criticalMultiplier) {
        // Змінна totalDamage "живе" тільки всередині цих фігурних дужок
        int totalDamage = baseAttack * criticalMultiplier;
        return totalDamage; // Повертаємо результат туди, де викликали метод
    }

    // Метод, який просто виводить текст (тому він void)
    public static void showGameOverScreen() {
        System.out.println("--- ГРА ЗАКІНЧЕНА ---");
    }
}
